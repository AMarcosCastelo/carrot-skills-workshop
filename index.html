Vamos começar com uma demo

Imagine que alguém escreveu este texto sobre a Carrot:

"A Carrot é uma startup inovadora que usa blockchain para salvar o planeta. Nossos clientes fazem parte de uma revolução verde que vai mudar tudo."

Vamos pedir ao Claude para revisar. No Claude Code Desktop com a pasta Carrot-AI:

"Revisa este texto e me diz se está alinhado com a voz da marca Carrot: 'A Carrot é uma startup inovadora que usa blockchain para salvar o planeta. Nossos clientes fazem parte de uma revolução verde que vai mudar tudo.'"

Observe o Claude ler as regras da marca, identificar os problemas e reescrever. Isso é um agente em ação.

DEMO AO VIVO ~3 min. O texto tem erros propositais: "startup" (é foundation), "salvar o planeta" (overclaim), "revolução" (agressivo), centra a marca (viola guardrails). O Claude vai ler brand.md, guardrails.md, flagrar cada erro e reescrever. O antes/depois é o "wow moment". Se der erro, não tem problema — mostre como lidar.

1. Conceitos essenciais

O mínimo para entender o que vamos usar

~10 min para este bloco. O objetivo é nivelar, não aprofundar.

Chat vs Agente — a distinção que importa

Chat / Assistente

  • Você pergunta, ele responde
  • Você guia cada passo
  • Não age sozinho — você copia o resultado

Ex: Claude Chat, ChatGPT, Gemini

Agente

  • Você dá um objetivo, ele resolve
  • Planeja, usa ferramentas, executa ações
  • Lê e edita arquivos, acessa Notion, etc.

Ex: Claude Code, Codex, Cursor (Agent)

Na prática: Um chat é como mandar mensagem para alguém pedindo ajuda. Um agente é como delegar uma tarefa — você diz o que quer e ele faz.
Este é o conceito central. Se a audiência entender só uma coisa, é esta. A demo anterior é o exemplo perfeito de agente.

Ferramentas de IA que usamos

O cenário

  • Claude (Anthropic) — principal na Carrot
  • ChatGPT (OpenAI) — disponível
  • Gemini (Google) — disponível
  • Cursor — editor com IA

Quando usar o quê?

  • Perguntas rápidas, tradução, resumos? Qualquer chat serve: ChatGPT, Gemini, Claude Chat
  • Precisa de contexto da Carrot, ferramentas ou ações? Claude Code

Todas as ferramentas estão disponíveis. Claude Code é a principal porque entende a Carrot e conecta com nossas ferramentas.

Não se demore aqui — 1 min. O ponto é: para coisas simples, use qualquer uma. Para coisas da Carrot, Claude Code.

Limitações — o que a IA não faz bem

Cuidado com

  • Alucinações — pode inventar fatos com confiança
  • Sem memória entre chats — cada conversa começa do zero
  • Conhecimento limitado — pode não saber eventos recentes

Como lidar

  • Sempre verifique dados importantes
  • Para informações da Carrot, ele usa as ferramentas (MCPs)
  • Decisões finais — especialmente compliance e comunicação — são suas
Regra de ouro: O Claude é um assistente, não uma fonte de verdade. Quanto mais importante a informação, mais você deve verificar.
Essencial para calibrar expectativas logo no início. Evita frustração depois.

2. Claude Code

O essencial para começar a usar

~10 min. Só o que eles precisam para o Hands-on 1. Detalhes (modelos, modos) vêm depois.

O ecossistema Claude

Claude Chat

Chat na web (claude.ai). Para perguntas rápidas, conversas gerais. Todos podem usar.

Claude Code

Agente completo. Acessa arquivos, usa ferramentas (Notion, Docs), segue regras da marca. A ferramenta principal da Carrot.

Claude Code roda na Desktop App (recomendado para a equipe), no CLI (engenharia) e na web. As três usam o mesmo modelo — a diferença é só a interface.
Simplifique: Chat = pergunta/resposta. Code = delega tarefas. Desktop App é o caminho para não-engenheiros.

Projetos — como o Claude "entende" contexto

Um projeto é uma pasta que o Claude Code abre e "entende".

O projeto Carrot-AI dá ao Claude:
  • Contexto sobre a Carrot (missão, produtos, marca)
  • Regras de comunicação e voz da marca
  • Acesso a ferramentas (Notion, Carrot Docs)
  • Comandos prontos (/setup, /help)
Para conversas simples (sem contexto da Carrot), use o Claude Chat em claude.ai.
Mostre na tela como abrir um projeto na Desktop App — aponte para a pasta Carrot-AI no Google Drive.

Conceitos que vamos usar agora

Prompt

A mensagem que você envia ao Claude. Pode ser uma pergunta, um pedido ou uma instrução.

MCP

Model Context Protocol — conecta o Claude a ferramentas externas (Notion, Carrot Docs, etc.).

Token

A unidade que o modelo processa. ~1 token ≈ 1 palavra. Conversas longas usam mais tokens.

Context Window

A "memória" do Claude em uma conversa. Tem limite — quando esgota, mensagens antigas são resumidas e podem perder detalhes.

Não precisa decorar. Vamos ver tudo isso em ação agora.

Rápido — 1 min. Esses termos vão aparecer no /context e /setup. Definir antes para que não sejam obstáculos.

Hands-on 1

Setup e exploração — vamos abrir juntos

~10 min. Faça junto com a audiência. Vá devagar. Pergunte "Quem já abriu? Levanta a mão." a cada passo.

Passo 1 — Abrir o projeto

  1. Abra o Claude Code Desktop
  2. Abra a pasta Carrot-AI do Google Drive como projeto
  3. Escreva: /setup
  4. Veja o resultado — quais ferramentas estão disponíveis?
Problemas? Levante a mão — vamos resolver juntos. Enquanto isso, olhe a tela do facilitador.
FACILITADOR: faça /setup na sua tela enquanto eles fazem. Circule pela sala. Se alguém travar, peça para um vizinho ajudar. Checkpoint: "Quem viu a lista de ferramentas?"

Passo 2 — Explorar o contexto

  1. Escreva: /help — veja o que o Claude pode fazer neste projeto
  2. Escreva: /context — veja o que está carregado
  3. Observe: quais rules, skills e MCPs aparecem?
O /context mostra o uso de tokens, ferramentas conectadas e instruções ativas. Você vai usar esse comando sempre que quiser entender o estado do Claude.
FACILITADOR: mostre o output do /context na sua tela. Aponte para: "Olha aqui os MCPs — Notion, Carrot Docs. Olha aqui os tokens — 8% de uso, bastante espaço." Checkpoint: "Todos viram o /context?"

Passo 3 — Modos e modelos

  1. No rodapé, clique no seletor de modo
  2. Veja as opções: Ask permissions (recomendado), Auto accept, Plan mode
  3. Volte para Ask permissions
  4. Clique no seletor de modelo — veja Sonnet, Opus, Haiku
  5. Confira o nível de esforço: Low / Medium / High
Configuração recomendada: Ask permissions + Sonnet + High. Só mude se precisar.
FACILITADOR: mostre cada clique na sua tela. 2 min max. Não precisa explicar todos os modelos — só o essencial.

3. Comunicar com IA

Como escrever bons pedidos — antes de praticar

~8 min. Este bloco prepara para o Hands-on 2. Ensine prompting ANTES de pedir que escrevam prompts.

Português ou Inglês?

Português

  • Mais natural para a maioria da equipe
  • Claude responde em PT automaticamente
  • Perfeitamente válido para tarefas do dia a dia

Inglês

  • Modelos treinados majoritariamente em EN
  • Termos técnicos mais precisos
  • Ligeiramente mais eficiente em tokens
Recomendação: Use a língua em que você se sente mais confortável. A diferença é mínima. Termos técnicos (skill, agent, MCP, token) ficam em inglês.
1 min. Resposta curta: use o que for natural para você.

Como escrever bons prompts

Menos eficaz

"Escreve um texto sobre reciclagem"

"Faz uma apresentação"

"Me ajuda com o marketing"

Mais eficaz

"Escreve um post de LinkedIn sobre o programa BOLD, focado em compradores corporativos, máximo 150 palavras"

"Cria uma apresentação de 5 slides sobre a Carrot para um investidor"

"Sugere 3 ideias de campanha para o Dia Mundial do Meio Ambiente"

Regra simples: Dê contexto + especifique o resultado. Não precisa ser perfeito — pode refinar depois.
2 min. Leia os exemplos em voz alta. Peça à audiência: "Notem a diferença: o lado esquerdo é vago, o direito tem contexto, público e formato."

Preciso escrever prompts perfeitos?

Não.

  • Erros de escrita? — O Claude entende mesmo assim
  • Frases incompletas? — Funciona
  • Misturar PT e EN? — Sem problema
  • Não saber o termo técnico? — Descreva o que você quer
O que importa: Ser específico sobre o que você quer, não como escreve. "post LinkedIn BOLD 150 palavras comprador" funciona tão bem quanto uma frase completa.
1 min. Isso é importante para reduzir a ansiedade de performance. Muita gente trava achando que precisa escrever o prompt "certo".

Peça ajuda para estruturar

Não sabe por onde começar? Peça ao Claude para ajudar a organizar as ideias antes de executar.

Exemplo: "Preciso criar uma campanha de marketing para o BOLD. Me ajuda a estruturar o que precisa ser feito?"

O Claude vai:

  • Fazer perguntas para entender o contexto
  • Sugerir uma estrutura e abordagens
  • Propor um plano antes de executar

Dica avançada: use o comando /brainstorming para um processo guiado de ideação.

2 min. Essa slide é chave para não-técnicos: você não precisa saber o que pedir. Peça ajuda para descobrir.

Hands-on 2

Tarefa real — vamos criar algo juntos

~15 min. Esta é a parte principal do workshop. Não apresse.

Tarefa compartilhada — todos juntos

Agora vamos ver o Claude buscar informação, não só gerar texto. Todos escrevam:

"Busca na documentação da Carrot o que é dMRV e me explica de forma simples em 3 bullets, como se eu fosse explicar para um comprador que nunca ouviu falar"

  1. Copie o prompt acima e envie ao Claude
  2. Observe: ele vai usar a ferramenta Carrot Docs (MCP) para buscar a informação
  3. Quando terminar, leia os 3 bullets
Repare na diferença: Na demo inicial, o Claude usou as regras da marca. Agora ele está buscando informação em uma ferramenta externa. São capacidades complementares.
FACILITADOR: ~4 min. Este é o "level 2" — a audiência vê o Claude chamando o MCP de Carrot Docs. Aponte na sua tela: "Olha, ele está buscando na documentação — não é algo que ele 'sabe', ele está indo buscar." Checkpoint: "Quem viu o Claude acessar a documentação?"

Compare com seu vizinho

Vire para a pessoa ao lado e compartilhe:

Discutam

  • A explicação ficou clara? Vocês usariam com um comprador?
  • O Claude foi buscar na documentação ou inventou?
  • O que vocês mudariam?

Experimente refinar

  • "Simplifica mais" ou "Agora faz uma versão em inglês"
  • "Adiciona um exemplo concreto de como o dMRV funciona"
  • Iterar é o poder do agente — não precisa acertar de primeira
FACILITADOR: 3 min para comparar. Pergunte: "Alguém pediu algo diferente ao refinar? O que?" Colete 2-3 respostas. Reforce: "Viram como é fácil refinar? Vocês não precisam acertar de primeira."

Exploração livre — sua vez

Agora experimente algo da sua área. Algumas ideias:

Conteúdo: "Escreve um post de LinkedIn sobre o BOLD, 150 palavras, para compradores corporativos"

Ferramentas (Notion): "Busca no Notion as últimas notas de reunião da minha equipe"

Marca: "Qual é a diferença entre community-built e community-led?"

Pesquisa: "Busca na documentação da Carrot sobre recycling credits e me faz um resumo"

Estruturar: "Preciso preparar uma apresentação sobre X. Me ajuda a estruturar?"

Tem 5 minutos. Explore e depois compartilhamos os resultados.

FACILITADOR: 5 min. Circule e observe o que as pessoas pedem — isso revela os casos de uso reais da equipe. Nos últimos 2 min: "Quem quer compartilhar?" Colete 2-3 exemplos.

4. A pasta Carrot-AI

O que está por trás do que vocês acabaram de usar

~5 min. Agora que eles já usaram, explicar o "por baixo" faz mais sentido.

O que é a pasta Carrot-AI?

A pasta no Google Drive que contém tudo o que o Claude precisa para ajudar a equipe.

O que vocês acabaram de usar

  • Instruções — o Claude sabe sobre a Carrot
  • Regras da marca — por isso o tom do post ficou alinhado
  • Ferramentas — Notion, Carrot Docs conectados
  • Skills — /setup, /help, /brainstorming

Como funciona

  1. Você abre a pasta como projeto
  2. O Claude lê o CLAUDE.md
  3. Rules, MCPs e skills carregam automaticamente
  4. Você só conversa

Mantido pela engenharia. Usado por todos.

Conecte com a experiência deles: "Quando vocês pediram o post do BOLD e o Claude já sabia o tom da marca — foi porque essa pasta tem as regras configuradas."

Estrutura da pasta

Carrot-AI/
  CLAUDE.md                    ← Instruções principais
  .claude/
    settings.json              ← Configuração de ferramentas (MCPs)
    skills/                    ← Comandos: /setup, /help
    rules/                     ← Voz da marca, comunicação
  context/
    brand/                     ← Brand guide completo
    company.md                 ← Missão, produtos, equipe
  docs/                        ← Glossário, guia de início
  workspaces/                  ← Projetos por equipe
  presentation/                ← Esta apresentação
Você não precisa mexer em nada. Basta abrir a pasta e usar. A engenharia mantém a configuração.
1 min. Mostre na tela mas não se aprofunde. O objetivo é dar uma noção da organização, não ensinar a estrutura.

5. Dicas e próximos passos

Para continuar usando com confiança

~10 min. Referência rápida + call to action concreto.

Detalhes úteis

Instruções (sob o capô)

  • CLAUDE.md — manual principal do projeto
  • Rules — regras automáticas (marca, tom)
  • Skills — comandos reutilizáveis (/setup, /help)
  • MCPs — conexões a ferramentas externas

Tudo configurado pela engenharia. Você só usa.

Quando começar um novo chat?

  • Continue — mesmo assunto, refinando resultado
  • Novo chat — mudou de tema, +50% de tokens no /context, Claude confuso

Regra prática: um assunto = um chat.

2 min. Referência rápida. Não precisa explicar cada item — eles já viram em ação.

Erros comuns e troubleshooting

Evite

  • Pedidos vagos ("me ajuda")
  • Colar senhas ou API keys
  • Confiar sem verificar
  • Conversas muito longas

Resolva

  • Resultado ruim? Seja mais específico ou refine
  • Informação errada? Peça fontes, verifique
  • Claude confuso? Novo chat resolve
  • Ferramentas sumindo? /context para diagnosticar
2 min. Formato lado a lado é mais fácil de escanear que uma tabela de 7 linhas.

Tokens — uso e economia

Cada mensagem consome tokens. Quanto mais longa a conversa, mais tokens são usados.

Boas práticas

  • Seja conciso nos pedidos (mas não vago)
  • Um assunto = um chat
  • Use Sonnet para tarefas simples
  • Não repita contexto que o Claude já tem

Hora de um novo chat

  • /context mostra mais de 50% de uso
  • Respostas ficam mais lentas ou genéricas
  • Você mudou de assunto
1 min. Rápido. Eles vão aprofundar em uma sessão futura.
