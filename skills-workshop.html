Como o Claude sabe quando usar

1. Você chama

  • Digita /carrot-voice no chat
  • Autocomplete mostra a skill
  • Enter — skill é invocada

2. Claude decide sozinho

  • Você escreve "revisa esse texto pra mim"
  • Claude lê o description da skill
  • Se bater com o pedido, invoca automaticamente
A description é o que faz Claude decidir. Quanto mais específica, melhor ele escolhe. "Revisa texto aplicando voz da Carrot" é muito melhor que "Skill de texto".

Skills podem ler outros arquivos

A parte mais poderosa. A skill do cold open referenciou o brand.md e o glossary.md — isso é o que permitiu ela revisar no tom Carrot sem a gente precisar copiar as regras pro arquivo dela.

Metáfora: é como se a skill dissesse "antes de me executar, vai ler esses arquivos". Você não precisa colar o contexto — só aponta pra onde ele já está.

Exemplos que você pode referenciar hoje

  • .claude/rules/brand.md — regras de voz e tom da Carrot
  • context/brand/glossary.md — termos e nomenclatura oficial
  • context/company.md — o que a Carrot é, missão, produtos
  • Qualquer arquivo seu no workspace
Slide 9 — o super-poder (~3 min). OBJETIVO: essa é a feature MATADORA. A skill do cold open só funcionou porque leu o brand.md — sem copiar o conteúdo. Pontos: - A skill apontou pra .claude/rules/brand.md e foi lá ler antes de executar - Metáfora: "É como apontar pra um livro que você já tem em casa, em vez de ter que copiar o livro inteiro toda vez" - Destacar o tip box Exemplos universais (ler rápido): - .claude/rules/brand.md — regras de voz - context/brand/glossary.md — termos oficiais - context/company.md — missão e produtos - Qualquer arquivo seu no workspace FECHAMENTO: "Isso é o que permite skills serem pessoais MAS consistentes com o tom da empresa. Cada um faz a sua, todas falam a mesma voz."

Skills podem até rodar código

Skills não precisam ser só texto. Elas podem rodar scripts — Python, shell, qualquer coisa — e usar o resultado na resposta.

Exemplos reais: skill que gera visualização do codebase, skill que busca dados no banco, skill que chama uma API externa.

Hoje a gente foca em skills só com texto — é 80% do valor e é o que dá pra aprender em 1h. Mas saiba que dá pra ir muito além quando você se sentir confortável.
Slide 10 — menção rápida (~1 min, NÃO MAIS). Objetivo: plantar a semente de que skills podem ir muito além de texto, sem entrar em detalhe. Falar rápido: "Skills podem rodar scripts: Python, shell, qualquer coisa. Gerar visualizações, buscar dados, chamar APIs." "Hoje a gente foca em texto — é 80% do valor. Mas saibam que existe esse outro nível quando vocês se sentirem confortáveis." NÃO demonstrar. NÃO dwell. Avançar.

1. Mais simples Pede pro /skill-creator criar pra você.

2. Intermediário Cria a pasta manualmente em ~/.claude/skills/nome/ e escreve o SKILL.md no seu editor de texto favorito.
Útil quando você já sabe o que quer e prefere escrever direto.
3. Avançado Cria skill de projeto em .claude/skills/ dentro do workspace Carrot-AI — pra compartilhar com o time inteiro.
Só Engineering commita no compartilhado, mas qualquer um pode pedir pra virar skill de projeto.
Hoje vocês vão usar o caminho 1. É o mais rápido pra ver a mágica acontecer.

Boas práticas rápidas

  • Descrição específica — não "skill de texto", mas "revisa texto aplicando voz da Carrot"
  • Começa simples — prefere 5 linhas que funcionam a 50 linhas quebradas
  • Testa com exemplo real — cola um texto de verdade e vê se funciona
  • Referencia contexto que já existe — brand.md, glossary.md, company.md
  • Se repetir 3+ vezes, vira skill — é a regra mais simples
Slide 12 — boas práticas (~2 min). 5 regras, ler cada uma com ênfase: 1. Descrição específica — "revisa texto aplicando voz da Carrot" é muito melhor que "skill de texto" 2.
